```python
import pytest
from agent_v2 import EnhancedAgent, TaskTracker

import shutil

def agent():

def test_task_tracker():

    assert tracker.get_pending_count() == 1
    tracker.complete(1)
    assert tracker.get_pending_count() == 0

    assert "too large" in result.lower()

    # Test normal safe edit (dry-run)
    result = agent._safe_self_edit("# Safe comment", dry_run=True)
```

Your agent now has full Docker deployment. **✅ Ultra-Optimized Distroless Dockerfile for Larry G-FORCE**

This version uses **distroless** for the runtime stage (extremely small, secure, no shell) while maximizing **layer caching** for fast rebuilds.

### Final `Dockerfile`

```dockerfile
# =============================================
# Larry G-FORCE — Ultra-Optimized Multi-Stage Distroless Dockerfile
# =============================================

# ==================== BUILDER STAGE ====================
FROM python:3.12-slim AS builder

WORKDIR /app

# Install only necessary build tools
RUN apt-get update && apt-get install -y --no-install-recommends \
    git \
    curl \
    build-essential \
    && rm -rf /var/lib/apt/lists/*

# Copy requirements first → best caching
COPY requirements.txt requirements-dev.txt ./

# Create venv and install dependencies
RUN python -m venv /app/venv \
    && /app/venv/bin/pip install --no-cache-dir --upgrade pip \
    && /app/venv/bin/pip install --no-cache-dir -r requirements.txt \
    && /app/venv/bin/pip install --no-cache-dir GitPython

# Copy application code (this layer changes most often)
COPY . /app/

# Make scripts executable
RUN chmod +x run-agent.sh

# ==================== RUNTIME STAGE (Distroless) ====================
FROM gcr.io/distroless/python3-debian12:latest

# Create non-root user
USER nonroot:nonroot

WORKDIR /app

# Copy only the venv from builder (biggest size win)
COPY --from=builder --chown=nonroot:nonroot /app/venv /app/venv

# Copy application files
COPY --from=builder --chown=nonroot:nonroot /app/agent_v2.py /app/
COPY --from=builder --chown=nonroot:nonroot /app/run-agent.sh /app/
COPY --from=builder --chown=nonroot:nonroot /app/ ./

# Environment
ENV PATH="/app/venv/bin:$PATH"
ENV PYTHONUNBUFFERED=1
ENV OLLAMA_URL=http://ollama:11434
ENV PYTHONPATH="/app"

# Persistent data volume
VOLUME ["/app/data"]

EXPOSE 8000

# Healthcheck
HEALTHCHECK --interval=30s --timeout=10s --start-period=60s --retries=3 \
    CMD ["/app/venv/bin/python", "-c", "import requests; requests.get('http://localhost:11434/api/tags', timeout=5) or exit(1)"]

# Run as non-root
CMD ["./run-agent.sh"]
```

### Key Optimizations Applied

| Optimization                     | Benefit                                      |
|----------------------------------|----------------------------------------------|
| Requirements copied first        | Maximum layer cache reuse                    |
| Distroless runtime               | ~150–250 MB final image (very small)         |
| Non-root user (`nonroot`)        | Better security                              |
| Venv copied from builder         | No redundant pip install in runtime          |
| Healthcheck                      | Monitors Ollama connectivity                 |
| Minimal runtime packages         | No shell, no unnecessary tools               |

### 2. Ollama Deployment + Service

**`02-ollama.yaml`**
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: ollama
  namespace: larry
spec:
  replicas: 1
  selector:
    matchLabels:
      app: ollama
  template:
    metadata:
      labels:
        app: ollama
    spec:
      containers:
      - name: ollama
        image: ollama/ollama:0.3.12
        ports:
        - containerPort: 11434
        volumeMounts:
        - name: ollama-data
          mountPath: /root/.ollama
        resources:
          requests:
            cpu: "1"
            memory: "2Gi"
          limits:
            cpu: "4"
            memory: "8Gi"
            nvidia.com/gpu: 1   # Adjust if you have multiple GPUs
        readinessProbe:
          httpGet:
            path: /api/tags
            port: 11434
          initialDelaySeconds: 30
          periodSeconds: 10
        livenessProbe:
          httpGet:
            path: /api/tags
            port: 11434
          initialDelaySeconds: 60
          periodSeconds: 30

      volumes:
      - name: ollama-data
        persistentVolumeClaim:
          claimName: ollama-data

---
apiVersion: v1
kind: Service
metadata:
  name: ollama
  namespace: larry
spec:
  selector:
    app: ollama
  ports:
  - port: 11434
    targetPort: 11434
  type: ClusterIP
```

### 4. Larry Agent Deployment

**`04-agent.yaml`**
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: larry-agent
  namespace: larry
spec:
  replicas: 1
  selector:
    matchLabels:
      app: larry-agent
  template:
    metadata:
      labels:
        app: larry-agent
    spec:
      containers:
      - name: agent
        image: larry-agent:latest   # built from your Dockerfile
        imagePullPolicy: IfNotPresent
        command: ["./run-agent.sh"]
        stdin: true
        tty: true
        env:
        - name: OLLAMA_URL

        resources:

            memory: "1Gi"
          limits:
            cpu: "2"
            memory: "4Gi"

          exec:
            command: ["/app/venv/bin/python", "-c", "import requests; requests.get('http://ollama.larry.svc.cluster.local:11434/api/tags', timeout=5) or exit(1)"]

          periodSeconds: 15
        livenessProbe:
          exec:
            command: ["/app/venv/bin/python", "-c", "import requests; requests.get('http://ollama.larry.svc.cluster.local:11434/api/tags', timeout=5) or exit(1)"]
```
