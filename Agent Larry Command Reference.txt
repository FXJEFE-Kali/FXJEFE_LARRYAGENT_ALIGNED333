  ---
  Security Tools

  ┌─────────────────────────────────┬─────────────────────────────────────────────────────┐
  │             Command             │                     Description                     │
  ├─────────────────────────────────┼─────────────────────────────────────────────────────┤
  │ /tools [category]               │ List all available security tools                   │
  ├─────────────────────────────────┼─────────────────────────────────────────────────────┤
  │ /kali list                      │ Same as /tools                                      │
  ├─────────────────────────────────┼─────────────────────────────────────────────────────┤
  │ /kali help <tool>               │ Show presets and usage for a tool                   │
  ├─────────────────────────────────┼─────────────────────────────────────────────────────┤
  │ /kali <tool> [args]             │ Run a security tool                                 │
  ├─────────────────────────────────┼─────────────────────────────────────────────────────┤
  │ /kali <tool> :<preset> <target> │ Run with a preset e.g. /kali nmap :quick 10.0.0.1   │
  ├─────────────────────────────────┼─────────────────────────────────────────────────────┤
  │ /<toolname> [args]              │ Shortcut — e.g. /nmap 10.0.0.1 /nikto http://target │
  └─────────────────────────────────┴─────────────────────────────────────────────────────┘

  Natural language also works for security tools — no slash needed:
  run nmap 10.0.0.1
  test dig google.com
  nmap -sV 192.168.1.1
  Rules: verb must be at the start, OR the tool name alone/with flags. Casual mentions like "nmap is great" are ignored.

  ---
  Tips

  - Presets speed up common scans: /nmap :quick target /nmap :service target /gobuster :dir http://target
  - --yes flag required to actually write file changes (safety preview by default)
  - /model auto re-enables smart routing after locking to a model
